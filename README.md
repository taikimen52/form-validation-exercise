課題：
    現状、バリデーションを行う関数内で状態管理とメッセージ内容の管理を分けていない。
    validateField内ではバリデーションチェックののち、エラーメッセージを直接書き換えており、
    表示はcssの擬似セレクタに任せている。

    バリデーションを行った後、is-invalid、is-validクラスを設定、エラーメッセージは変数に保存して、表示用の関数に渡す方がクリーン。

    cssは最低限のスタイルしかあてておらず、エラーメッセージが表示されると次のフィールドが下にズレるので、positionの設定とマージンの設定をするといい。
